# セキュリティチェックリスト

このチェックリストは `--append-system-prompt-file` 経由で Claude Code に注入される。
レビュー対象のコードに対して、以下の全項目を網羅的にチェックすること。

## OWASP Top 10 (2021)

### A01: アクセス制御の不備
- 認証バイパスの可能性（認証ミドルウェアの適用漏れ）
- 水平権限昇格（他ユーザーのデータへのアクセス、IDOR）
- 垂直権限昇格（管理者機能への不正アクセス）
- CORS の設定ミス（ワイルドカード `*` の使用）
- パストラバーサル（`../` を含むパスの未検証）

### A02: 暗号化の失敗
- 平文での機密情報の保存・送信（パスワード、トークン、API キー）
- 弱い暗号アルゴリズム（MD5, SHA-1）の使用
- ハードコードされたシークレット（API キー、パスワード、暗号化キー）
- 不適切な証明書検証（TLS 検証の無効化）

### A03: インジェクション
- SQL インジェクション（パラメータ化されていないクエリ）
- コマンドインジェクション（ユーザー入力の `exec`, `spawn` への直接渡し）
- XSS（`innerHTML`, `dangerouslySetInnerHTML`, 未エスケープ出力）
- テンプレートインジェクション（テンプレートリテラルへのユーザー入力）

### A04: 安全でない設計
- ビジネスロジックの欠陥（レースコンディション、TOCTOU）
- レート制限の欠如（ブルートフォース攻撃の余地）
- 入力バリデーション不足（型チェック、範囲チェック）

### A05: セキュリティの設定ミス
- デフォルト認証情報の使用
- 詳細すぎるエラーメッセージ（スタックトレース、DB スキーマの漏洩）
- 不要な機能の有効化（デバッグモード、Swagger UI の本番公開）
- セキュリティヘッダーの欠如（CSP, X-Frame-Options, HSTS）

### A06: 脆弱で古いコンポーネント
- 既知の CVE を持つ依存関係
- サポート終了のライブラリ・フレームワーク

### A07: 認証と識別の失敗
- セッション管理の不備（セッション固定、無期限セッション）
- JWT の署名未検証、有効期限未チェック
- CSRF 対策の欠如

### A08: ソフトウェアとデータの整合性の失敗
- 安全でないデシリアライゼーション（`JSON.parse` の入力元未検証）
- CI/CD パイプラインの検証不足

### A09: セキュリティログとモニタリングの失敗
- 重要操作（認証失敗、権限変更）のログ記録不足
- ログへの機密情報出力（トークン、パスワード）
- 監査証跡の欠如

### A10: サーバーサイドリクエストフォージェリ (SSRF)
- ユーザー提供 URL の未検証 fetch/request
- 内部リソースへのアクセス制御の欠如（メタデータサービス等）

## 追加チェック項目

### 機密情報管理
- `.env` ファイルが `.gitignore` に含まれているか
- API キー・トークン・パスワードのハードコード
- 機密情報のログ出力（Bearer トークン、セッション ID）
- 環境変数経由でのシークレット管理

### フロントエンド固有
- `innerHTML` / `dangerouslySetInnerHTML` への未サニタイズ入力
- `eval()` / `Function()` の使用
- `postMessage` の origin 未検証
- `localStorage` / `sessionStorage` への機密情報保存

### API 固有
- 全エンドポイントへの認証ミドルウェア適用
- レスポンスへの内部情報漏洩（DB ID、内部パス）
- Content-Type の適切な検証
- ファイルアップロードのサイズ制限・拡張子検証

## 出力形式

各脆弱性について以下のフォーマットで報告すること:

### [重大度] 脆弱性タイトル
- **重大度**: Critical / High / Medium / Low / Info
- **ファイル**: `path/to/file.ts:行番号`
- **カテゴリ**: OWASP A01-A10 / その他
- **問題**: 何が問題か
- **影響**: 悪用された場合の影響
- **修正案**: 具体的な修正コードまたは対策

重大度の定義:
- **Critical**: リモートからの不正アクセス・データ漏洩が即座に可能
- **High**: 認証済みユーザーによる権限昇格・他データアクセスが可能
- **Medium**: 限定的な情報漏洩・条件付きの脆弱性
- **Low**: ベストプラクティスからの逸脱・将来的なリスク
- **Info**: 改善推奨事項（直接的なセキュリティリスクなし）
